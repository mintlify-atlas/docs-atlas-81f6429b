## Update Caregiver

<api method="PUT" endpoint="/api/caregivers" />

Updates an existing caregiver profile with new information.

### Authentication Required

This endpoint requires a valid JWT token.

<Note>
The caregiver `id` must be included in the request body to identify which profile to update.
</Note>

### Request

#### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  ```
  Authorization: Bearer <your_jwt_token>
  ```
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

#### Body Parameters

<ParamField body="id" type="string" required>
  The unique identifier of the caregiver profile to update
</ParamField>

<ParamField body="presentation" type="string">
  Updated text description of the caregiver's services and experience
</ParamField>

<ParamField body="state" type="number">
  Updated publication state
  
  **Possible values:**
  - `1` - PendingReview
  - `2` - Draft
  - `3` - Published
  - `4` - Archived
  - `5` - Rejected
  - `6` - Deleted
</ParamField>

<ParamField body="userId" type="string">
  User ID (note: updating this field may cause data inconsistency)
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message: "Ok"
</ResponseField>

<ResponseField name="data" type="object">
  The complete update object that was sent in the request (including the ID)
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code: `200`
</ResponseField>

### Examples

<CodeGroup>

```bash cURL
curl -X PUT https://api.rescuers.com/api/caregivers \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "id": "caregiver_abc123",
    "presentation": "Updated: Experienced animal caregiver now offering 24/7 emergency care services. Specialized in medical needs and senior pet care.",
    "state": 3
  }'
```

```javascript JavaScript
const response = await fetch('https://api.rescuers.com/api/caregivers', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    id: 'caregiver_abc123',
    presentation: 'Updated: Experienced animal caregiver now offering 24/7 emergency care services. Specialized in medical needs and senior pet care.',
    state: 3
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

url = 'https://api.rescuers.com/api/caregivers'
headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
}
payload = {
    'id': 'caregiver_abc123',
    'presentation': 'Updated: Experienced animal caregiver now offering 24/7 emergency care services. Specialized in medical needs and senior pet care.',
    'state': 3
}

response = requests.put(url, json=payload, headers=headers)
print(response.json())
```

## Delete Caregiver

<api method="DELETE" endpoint="/api/caregivers/:id" />

Permanently deletes a caregiver profile from the system.

### Authentication Required

This endpoint requires a valid JWT token.

<Warning>
This operation permanently deletes the caregiver profile. This action cannot be undone. Consider using the update endpoint to set the state to `Archived` (4) or `Deleted` (6) instead if you want to preserve the data.
</Warning>

### Request

#### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  ```
  Authorization: Bearer <your_jwt_token>
  ```
</ParamField>

#### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the caregiver profile to delete
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message: "Ok"
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code: `200`
</ResponseField>

### Examples

<CodeGroup>

```bash cURL
curl -X DELETE https://api.rescuers.com/api/caregivers/caregiver_abc123 \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## Soft Delete Alternative

Instead of permanently deleting a caregiver profile, you can use the UPDATE endpoint to set the state to archived or deleted:

```bash
curl -X PUT https://api.rescuers.com/api/caregivers \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "caregiver_abc123",
    "state": 6
  }'
```

This preserves the data while marking it as deleted (state = 6) or archived (state = 4).
