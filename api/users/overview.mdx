## Introduction

The Users API provides endpoints for managing user accounts, profiles, and retrieving user information. This API group handles user data operations including retrieval, updates, and file uploads.

## Base URL

All user endpoints are accessible under:

```
/api/auth
```

## Authentication

Most user management endpoints require authentication via JWT token. Include the token in the Authorization header:

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

### Endpoints Requiring Authentication:
- `PUT /api/auth/` - Update user
- `DELETE /api/auth/:id` - Delete user
- `POST /api/auth/uploads` - Upload user image
- `GET /api/auth/roles` - Get user roles

### Public Endpoints:
- `GET /api/auth/` - Get all users
- `GET /api/auth/:id` - Get user by ID
- `GET /api/auth/byemail/:email` - Get user by email

## User Object

The User object contains the following fields:

<ResponseField name="id" type="string">
  Unique identifier for the user
</ResponseField>

<ResponseField name="name" type="string" required>
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string" required>
  User's last name
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address (unique)
</ResponseField>

<ResponseField name="image" type="string">
  URL to user's profile image
</ResponseField>

<ResponseField name="contacts" type="array">
  Array of contact information objects
  
  <Expandable title="Contact Object">
    <ResponseField name="contact" type="string">
      Contact value (phone number, email, etc.)
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Type of contact (e.g., "phone", "mobile", "email")
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="roles" type="array">
  Array of role IDs assigned to the user
</ResponseField>

<ResponseField name="emailConfirmed" type="boolean">
  Whether the user's email has been verified
</ResponseField>

<ResponseField name="emailVerificationAttempts" type="number">
  Number of email verification attempts
</ResponseField>

<ResponseField name="zoneId" type="string">
  ID of the zone associated with the user
</ResponseField>

## Error Responses

All endpoints return standard error responses:

<ResponseField name="message" type="string">
  Human-readable error message
</ResponseField>

<ResponseField name="code" type="number">
  HTTP status code
</ResponseField>

<ResponseField name="state" type="string">
  Error state identifier
</ResponseField>

### Common Error Codes

| Code | Description |
|------|-------------|
| 400  | Bad Request - Invalid input data |
| 401  | Unauthorized - Invalid or missing authentication token |
| 404  | Not Found - User does not exist |
| 500  | Internal Server Error |
