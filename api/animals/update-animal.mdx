---
title: 'Update Animal'
api: 'PUT /api/animals'
description: 'Update an existing animal rescue post'
---

## Authentication

<Note>
  This endpoint requires authentication. Include a valid Bearer token in the Authorization header.
</Note>

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

## Request Body

<ParamField body="id" type="string" required>
  The unique identifier of the animal to update
</ParamField>

<ParamField body="name" type="string">
  The name of the animal
</ParamField>

<ParamField body="description" type="string">
  Detailed description of the animal
</ParamField>

<ParamField body="state" type="number">
  Current state of the post:
  - `1` - Pending Review
  - `2` - Draft
  - `3` - Published
  - `4` - Archived
  - `5` - Rejected
  - `6` - Deleted
</ParamField>

<ParamField body="lost" type="boolean">
  Indicates whether the animal is lost (`true`) or available for adoption/rescue (`false`)
</ParamField>

<ParamField body="atributes" type="string[]">
  Array of attributes describing the animal
</ParamField>

<ParamField body="image" type="string">
  URL of the animal's image
</ParamField>

<ParamField body="userId" type="string">
  ID of the user who created the post (usually shouldn't be modified)
</ParamField>

<Note>
  You only need to include the fields you want to update along with the required `id` field. All other fields are optional.
</Note>

## Response

Returns the updated animal object with all provided fields.

<ResponseField name="id" type="string">
  Unique identifier for the animal
</ResponseField>

<ResponseField name="name" type="string">
  Updated name of the animal
</ResponseField>

<ResponseField name="userId" type="string">
  ID of the user who created the post
</ResponseField>

<ResponseField name="image" type="string">
  Updated URL of the animal's image
</ResponseField>

<ResponseField name="description" type="string">
  Updated description of the animal
</ResponseField>

<ResponseField name="state" type="number">
  Updated state of the post
</ResponseField>

<ResponseField name="lost" type="boolean">
  Updated lost status
</ResponseField>

<ResponseField name="atributes" type="string[]">
  Updated array of animal attributes
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X PUT https://api.rescuers.com/api/animals \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "id": "abc123xyz",
    "name": "Max (Updated)",
    "description": "Friendly golden retriever looking for a loving home. Recently trained and great with kids.",
    "state": 3
  }'
```

```javascript JavaScript
const response = await fetch('https://api.rescuers.com/api/animals', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  },
  body: JSON.stringify({
    id: 'abc123xyz',
    name: 'Max (Updated)',
    description: 'Friendly golden retriever looking for a loving home. Recently trained and great with kids.',
    state: 3
  })
});

const data = await response.json();
```

```python Python
import requests

url = 'https://api.rescuers.com/api/animals'
headers = {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
}
data = {
    'id': 'abc123xyz',
    'name': 'Max (Updated)',
    'description': 'Friendly golden retriever looking for a loving home. Recently trained and great with kids.',
    'state': 3
}

response = requests.put(url, json=data, headers=headers)
result = response.json()
```

</CodeGroup>

## Example Response

```json 200 - Success
{
  "id": "abc123xyz",
  "name": "Max (Updated)",
  "description": "Friendly golden retriever looking for a loving home. Recently trained and great with kids.",
  "state": 3
}
```

```json 401 - Unauthorized
{
  "error": "Unauthorized",
  "details": {}
}
```

```json 404 - Not Found
{
  "error": "Animal not found",
  "details": {}
}
```

```json 500 - Internal Server Error
{
  "error": "Error al actualizar",
  "details": {}
}
```

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Animal updated successfully |
| 400 | Bad request - invalid or missing id field |
| 401 | Unauthorized - missing or invalid authentication token |
| 404 | Animal not found |
| 500 | Internal server error |

## Notes

<Note>
  When updating an animal post, the response will only include the fields that were sent in the request. To get the complete updated object, you may need to make a subsequent GET request to `/api/animals/:id`.
</Note>
