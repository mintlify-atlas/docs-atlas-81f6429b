## Overview

This endpoint allows clients to obtain a new access token without requiring the user to log in again. When the short-lived access token expires, use the long-lived refresh token to get a new set of tokens.

<Note>
  After a successful refresh, the old refresh token is invalidated and a new one is issued. Always store the new refresh token for future use.
</Note>

## Request

### Body Parameters

<ParamField body="refreshToken" type="string" required>
  The refresh token obtained from the login or previous refresh operation. This is a UUID v4 string.
</ParamField>

## Response

### Success Response (200)

<ResponseField name="token" type="string">
  New JWT access token for authenticating API requests. Expires in 10 seconds.
</ResponseField>

<ResponseField name="sub" type="string">
  Subject identifier - the user's unique ID
</ResponseField>

<ResponseField name="refreshToken" type="string">
  New refresh token (valid for 6 months). The old refresh token is now invalidated.
</ResponseField>

```javascript JavaScript
const refreshAccessToken = async () => {
  const refreshToken = localStorage.getItem('refreshToken');
  
  const response = await fetch('https://api.rescuers.com/api/auth/no-auth/refresh-token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ refreshToken })
  });

  if (response.ok) {
    const data = await response.json();
    
    // Update stored tokens
    localStorage.setItem('accessToken', data.token);
    localStorage.setItem('refreshToken', data.refreshToken);
    
    return data.token;
  } else {
    // Refresh failed - redirect to login
    window.location.href = '/login';
    return null;
  }
};

// Use with automatic retry on 401
const makeAuthenticatedRequest = async (url, options = {}) => {
  let token = localStorage.getItem('accessToken');
  
  let response = await fetch(url, {
    ...options,
    headers: {
      ...options.headers,
      'Authorization': `Bearer ${token}`
    }
  });
  
  // If token expired, refresh and retry
  if (response.status === 401) {
    token = await refreshAccessToken();
    if (token) {
      response = await fetch(url, {
        ...options,
        headers: {
          ...options.headers,
          'Authorization': `Bearer ${token}`
        }
      });
    }
  }
  
  return response;
};
```

```python Python
import requests

def refresh_access_token(refresh_token):
    response = requests.post(
        'https://api.rescuers.com/api/auth/no-auth/refresh-token',
        json={'refreshToken': refresh_token}
    )
    
    if response.status_code == 200:
        data = response.json()
        return {
            'access_token': data['token'],
            'refresh_token': data['refreshToken'],
            'user_id': data['sub']
        }
    else:
        return None

# Usage example
stored_refresh_token = get_stored_refresh_token()
result = refresh_access_token(stored_refresh_token)

if result:
    # Update stored tokens
    save_tokens(result['access_token'], result['refresh_token'])
else:
    # Redirect to login
    redirect_to_login()
```

## Error Responses

### Invalid or Expired Refresh Token (200)

<Note>
  Even when the refresh fails, the endpoint returns a 200 status code with a `null` response body. This is an implementation detail of the current API.
</Note>

```json
null
```

Possible reasons for a null response:
- Refresh token doesn't exist in the database
- Refresh token has expired (older than 6 months)
- User associated with the refresh token no longer exists
- Refresh token has already been used (single-use tokens)

## Implementation Details

<Note>
  **Source Reference:** `src/controllers/user/user.controller.ts:206-217` and `src/services/user/user.service.ts:314-336`
</Note>

### Refresh Flow

1. **Token Validation**: System looks up the refresh token in the database
2. **Expiry Check**: Validates that the token hasn't expired (< 6 months old)
3. **User Validation**: Ensures the user associated with the token still exists
4. **Token Rotation**: 
   - Deletes the old refresh token from the database
   - Generates a new access token (10-second expiry)
   - Generates a new refresh token (6-month expiry)
   - Stores the new refresh token in the database
5. **Response**: Returns new token pair and user ID

### Token Rotation Strategy

The API implements a **refresh token rotation** security pattern:

- Each refresh token is single-use only
- After a successful refresh, the old token is immediately invalidated
- A new refresh token is issued with each refresh operation
- This limits the window of opportunity if a refresh token is compromised

### Security Benefits

<Expandable title="Why Token Rotation?">
  Token rotation provides several security advantages:
  
  - **Breach Detection**: If an attacker uses a stolen refresh token, the legitimate user's next refresh attempt will fail, indicating a potential compromise
  - **Limited Lifetime**: Even if a refresh token is stolen, it becomes useless after the legitimate user refreshes
  - **Audit Trail**: Each refresh creates a new token, making it easier to track and audit authentication events
  - **Revocation**: Compromised tokens can be revoked without affecting other user sessions
</Expandable>

## Best Practices

### Client-Side Implementation

1. **Automatic Refresh**: Implement automatic token refresh when receiving 401 responses
2. **Secure Storage**: Store refresh tokens securely (HttpOnly cookies or secure storage)
3. **Update Tokens**: Always update both the access and refresh tokens after a successful refresh
4. **Handle Failures**: Redirect to login when refresh fails (indicates expired session)

### Token Lifecycle Management

```javascript
// Example: Token refresh before expiration
class TokenManager {
  constructor() {
    this.accessToken = null;
    this.refreshToken = null;
    this.refreshTimeout = null;
  }
  
  setTokens(accessToken, refreshToken) {
    this.accessToken = accessToken;
    this.refreshToken = refreshToken;
    
    // Schedule refresh before access token expires (10s - refresh at 8s)
    this.scheduleRefresh(8000);
  }
  
  scheduleRefresh(delay) {
    if (this.refreshTimeout) {
      clearTimeout(this.refreshTimeout);
    }
    
    this.refreshTimeout = setTimeout(async () => {
      const newTokens = await this.refresh();
      if (newTokens) {
        this.setTokens(newTokens.token, newTokens.refreshToken);
      }
    }, delay);
  }
  
  async refresh() {
    const response = await fetch('/api/auth/no-auth/refresh-token', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ refreshToken: this.refreshToken })
    });
    
    return response.ok ? await response.json() : null;
  }
}
```
