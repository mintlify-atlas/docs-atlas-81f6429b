## Overview

This endpoint allows new users to create an account. After successful registration, a confirmation email will be sent to the provided email address. The user must verify their email before being able to log in.

<Note>
  The registration process is a two-step flow:
  1. Register with this endpoint
  2. Confirm email using the token sent to the provided email address
</Note>

## Request

### Body Parameters

<ParamField body="email" type="string" required>
  User's email address. Must be unique in the system.
</ParamField>

<ParamField body="name" type="string" required>
  User's first name
</ParamField>

<ParamField body="lastName" type="string" required>
  User's last name
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Will be hashed using bcrypt before storage.
</ParamField>

<ParamField body="phone" type="string">
  User's phone number (optional). If provided, will be stored as a WhatsApp contact.
</ParamField>

## Response

<ResponseField name="statusCode" type="number">
  Response status code indicating the result of the registration attempt
  
  <Expandable title="Status Codes">
    - `200`: Registration successful, verification email sent
    - `290`: Email already registered and verified
    - `291`: Email already registered but not verified
    - `500`: Internal server error (e.g., failed to send confirmation email)
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message describing the registration result
</ResponseField>

## Implementation Details

<Note>
  **Source Reference:** `src/controllers/user/user.controller.ts:77-87`
</Note>

### User Data Structure

When a user is successfully registered, the following data is stored:

- Email, name, lastName, and hashed password
- Email confirmation token (16-character random string)
- Default role: `USER`
- Email confirmed status: `false`
- Email verification attempts: `0`
- Default zone ID: `3`
- Optional WhatsApp contact if phone is provided

### Security Features

- Passwords are hashed using bcrypt with a salt rounds of 10
- Email confirmation token is a randomly generated 16-character string
- If email sending fails, the user record is automatically deleted (rollback)

## Next Steps

After successful registration:

1. Check the email inbox for the verification email
2. Use the `/api/auth/no-auth/confirm-email` endpoint with the token from the email
3. Once verified, use the `/api/auth/no-auth/login` endpoint to authenticate

If the verification email wasn't received, use `/api/auth/no-auth/resend-verification-mail` to request a new one.
