## Overview

The Advertisements API allows you to create, read, update, and delete advertisements. Advertisements are promotional content items that can be displayed alongside animal listings.

**Base URL**: `https://your-api-domain.com/api/advertisements`

## Authentication

- **GET** endpoints are public (no authentication required)
- **POST, PUT, DELETE** endpoints do not require authentication in the current implementation

## Advertisement Object

The Advertisement model represents a promotional content item in the system.

```typescript IAdvertisement
interface IAdvertisement extends BaseModel {
  id: string;
  userId: string;
  content: string;
  publicationDate?: string;
  state: number;
}
```

<ResponseField name="id" type="string" required>
  Unique identifier for the advertisement
</ResponseField>

<ResponseField name="userId" type="string" required>
  ID of the user who created the advertisement
</ResponseField>

<ResponseField name="content" type="string" required>
  The advertisement content or message
</ResponseField>

<ResponseField name="publicationDate" type="string">
  ISO 8601 date string indicating when the advertisement was published
</ResponseField>

<ResponseField name="state" type="number" required>
  Current state of the advertisement (0=inactive, 1=active, etc.)
</ResponseField>

## Error Responses

All endpoints return standard error responses:

```json
{
  "error": "Error interno del servidor."
}
```

Common status codes:
- `200` - Success
- `201` - Created
- `500` - Internal server error
